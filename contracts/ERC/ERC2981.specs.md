# ERC2981.sol User Stories

## Scope
NFT royalty standard exposing royalty receiver and amount for secondary sales.

## Primary Actors
- Creator/royalty recipient
- NFT marketplace
- Collection admin

## User Stories
- As a creator, I want royalty info attached to token sales, so that I can receive secondary-sale revenue.
- As a marketplace, I want to query royalty receiver and amount, so that payouts can be computed consistently.
- As a collection admin, I want configurable royalty parameters, so that business terms can be managed.

## Acceptance Criteria
- Royalty query returns receiver and amount based on sale price.
- Royalty fractions are bounded by a denominator and cannot exceed logical limits.
- Default and token-specific royalty settings behave according to contract rules.
